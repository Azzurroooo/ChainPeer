# ChainPeer Windows 产品规划

## 1. 产品定位

ChainPeer 是一款面向金融研究、量化投资和专业知识工作的本地 AI Agent 产品。它底层是一个通用 agent，具备文件理解、代码执行、网络搜索、计划管理、技能扩展和长任务执行能力；但产品卖点会优先聚焦金融场景，尤其是因子检测、策略迭代、研报复现和投资分析。

产品形态参考 Cursor：

- 软件可以自由下载安装。
- 用户在本地电脑中运行。
- 通过订阅或 token 记账使用模型能力。
- 安装后可在终端中直接启动 agent。

ChainPeer 的核心价值是：

- 帮助用户把金融研究想法快速转化为可验证结果。
- 自动执行数据处理、回测、报告生成和策略迭代。
- 支持长时间任务，适合复杂策略优化。
- 保留本地文件和研究资产的控制权。

## 2. 产品形态

第一阶段只考虑 Windows 版本。

产品以独立安装包形式分发：

```text
ChainPeerSetup.exe
```

安装后，用户可以在任意终端运行：

```text
chainpeer
```

或简写：

```text
cp
```

启动后进入交互式 agent CLI。

### 2.1 使用入口

优先支持：

- Windows Terminal
- PowerShell
- Git Bash
- Cursor / VS Code 内置终端

### 2.2 Git Bash 支持

当前 bash 执行依赖 Git Bash，因此 Windows 安装需要处理 Git Bash 环境。

建议策略：

1. 如果用户已有 Git for Windows，则自动检测并使用。
2. 如果没有，则提示安装轻量 Git for Windows。
3. 后续可以提供内置最小 Git Bash runtime 的安装包。

## 3. 目标用户

### 3.1 个人量化研究者

他们需要快速验证因子、策略和市场假设，但不希望每次都从数据处理、回测框架、报告模板开始搭建。

### 3.2 金融从业者

包括研究员、投顾、交易员、资产配置人员。他们需要把研报、市场信息、持仓情况和历史数据结合起来，形成更快的分析和决策辅助。

### 3.3 有编程能力的投资者

他们懂一些 Python 或数据分析，但希望 AI 帮助完成重复工作、代码生成、策略调参和结果总结。

### 3.4 小型量化团队

他们希望获得一个本地优先、可扩展、可沉淀研究流程的 agent 工具，而不是完全依赖云端黑盒平台。

## 4. 核心金融场景

### 4.1 因子检测

用户输入因子表达式，以及想要验证的标的范围，ChainPeer 自动完成因子检测并生成报告。

典型输入：

```text
帮我检测这个因子：ROE / PB
标的是沪深300成分股，周期为过去5年
```

Agent 需要完成：

- 理解因子表达式。
- 读取或获取标的数据。
- 构建因子值。
- 运行 IC、分组收益、多空收益、换手率等检测。
- 生成稳定格式的因子检测报告。

为了保证报告稳定性，产品应内置一套标准化因子检测脚本和报告模板。Agent 可以调用这些脚本，而不是每次临时生成完全不同的分析代码。

### 4.2 策略迭代

用户提供基础策略或投资思路，ChainPeer 自动进行策略实现、回测、分析和迭代。

典型输入：

```text
我想做一个低波红利策略，目标是年化超过10%，夏普超过2，请你持续迭代。
```

Agent 需要完成：

- 将策略想法转化为初版可运行策略。
- 自动运行回测。
- 分析收益、回撤、换手、夏普等指标。
- 基于结果提出下一步优化方向。
- 继续改策略、跑回测、记录观察。
- 最终沉淀一个指标较好的策略版本。

这个场景可能是超长时间任务，持续数小时甚至数天。因此需要依赖：

- plan 长任务管理
- observation 记录
- session 持久化
- 自动恢复
- 稳定回测脚本
- 可追踪的实验日志

### 4.3 研报复现

用户提供研报 PDF，ChainPeer 解析研报内容，并尝试复现其中的因子、策略或研究结论。

典型输入：

```text
请复现这篇研报里的行业轮动策略。
```

Agent 需要完成：

- 解析 PDF。
- 提取策略逻辑、因子定义、样本范围和调仓规则。
- 将研报描述转化为可执行代码。
- 运行回测。
- 对比研报结论和本地复现结果。
- 生成复现报告。

该场景的卖点是把“读研报”和“写代码回测”连接起来，降低研究复现门槛。

### 4.4 投资建议

用户可以宽泛提问，也可以提供自己的持仓情况，ChainPeer 基于网络信息、市场数据和模型分析能力，提供进一步投资分析建议。

典型输入：

```text
我现在持有白酒、银行和新能源，帮我看看组合风险和近期需要关注的信息。
```

Agent 需要完成：

- 理解用户持仓。
- 搜索最新市场信息。
- 分析行业、估值、风险和事件。
- 给出组合层面的观察和建议。
- 明确不确定性和风险提示。

该功能应定位为投资研究辅助，不应承诺收益，也不应包装成确定性荐股工具。

## 5. 通用 Agent 能力

虽然产品卖点聚焦金融，但底层仍应保持通用能力。

核心能力包括：

- 项目文件理解
- 代码搜索与修改
- 终端命令执行
- Web 搜索与页面读取
- PDF / 文档解析
- Plan 长任务管理
- Skill 可扩展能力
- 会话持久化与恢复
- Tool 调用状态展示
- 基础安全审批
- Token / context 使用情况展示

金融能力可以通过内置脚本、模板和 skills 强化，但不应把系统做死成只能处理金融任务。

## 6. 产品体验

用户安装后的理想路径：

1. 下载 `ChainPeerSetup.exe`
2. 安装
3. 打开终端
4. 输入 `chainpeer`
5. 首次启动完成登录或 API key 配置
6. 进入项目目录或研究目录
7. 开始对话式研究

示例：

```text
cd my-research
chainpeer
```

进入后：

```text
> 帮我检测这个因子
> 根据这份研报复现策略
> 帮我优化这个回测结果
> 总结我当前持仓的风险
```

体验目标：

- 用户不需要理解内部 tool。
- 用户不需要手动搭建每个分析脚本。
- Agent 应主动展示当前在做什么。
- 长任务应能持续跟踪进度。
- 结果应以报告、图表、代码和日志形式沉淀。

## 7. 金融能力建设方式

为了让金融场景稳定可用，需要提前内置一批标准能力。

### 7.1 标准回测脚本

内置或生成固定结构的回测脚本，支持：

- 因子检测
- 分组回测
- 多空组合
- 单策略回测
- 指标汇总
- 图表输出

### 7.2 报告模板

保证输出格式稳定：

- 因子检测报告
- 策略回测报告
- 研报复现报告
- 投资组合分析报告

### 7.3 金融 Skills

可内置一批金融 skills：

- factor-analysis
- strategy-backtest
- report-replication
- portfolio-analysis
- risk-review

用户也可以创建自己的私有 skill。

### 7.4 长任务计划

策略迭代类任务必须使用 plan：

- 目标
- 指标
- 当前策略版本
- 最近一次回测结果
- 下一步假设
- 已失败的方法

这样即使任务跨越多轮对话或多天运行，也能保持连续性。

## 8. 商业模式

产品采用“免费安装，付费使用 AI 与金融增强能力”的商业模式。软件本体可以自由下载和安装，收入主要来自订阅套餐与超额 token 计费。

### 8.1 收费原则

- 安装免费，降低试用门槛。
- 核心模型调用按订阅和 token 额度限制。
- 高级金融 skills、内置脚本、报告模板和长任务能力按订阅等级开放。
- 重度用户可在订阅额度外继续按 token 付费。

### 8.2 Token 记账

Token 记账用于承接模型成本和长时间任务成本。产品需要提供清晰的 token 展示、消费记录和月度消费上限，尤其是在策略迭代这类可能持续数小时甚至数天的任务中。

具体订阅分层见第 13 节。

## 9. 账号与登录

第一阶段支持简单登录：

```text
/login
```

用户可以选择：

1. 官方托管模型服务
2. 自带 API key

自带 API key 适合早期开发者和高级用户。官方托管模型服务适合商业化订阅和 token 计费。

## 10. 分发策略

### 10.1 第一阶段

- 官网下载 Windows 安装包
- GitHub release 下载
- 检测 Git for Windows
- 安装后自动写入 PATH

### 10.2 第二阶段

- 安装包内置环境检查
- 自动更新
- 错误诊断包
- 更完整的金融模板与示例项目

### 10.3 第三阶段

- 企业离线安装包
- 团队配置
- 内网模型服务
- 私有数据源接入

## 11. 数据与隐私

产品应明确强调：

- 项目文件默认保留在用户本地。
- 会话记录保存在本机 `.chainpeer` 目录。
- 发送给模型的数据仅限完成任务所需上下文。
- 用户可以清理 session。
- 企业版本可支持更严格的数据策略。

金融用户对数据安全敏感，因此隐私承诺必须清晰。

## 12. 安全与合规

因为 agent 可以执行命令和生成投资分析，需要明确边界。

### 12.1 命令安全

至少支持：

- 高危命令拒绝
- 风险命令确认
- 普通只读命令允许

典型交互：

```text
Agent wants to run:
rm -rf output

Allow? [y/N]
```

### 12.2 投资建议边界

投资建议功能应明确定位为研究辅助。

需要避免：

- 承诺收益
- 确定性荐股
- 无风险表述
- 伪造数据来源

建议在产品中保留必要风险提示。

## 13. 订阅版本规划

ChainPeer 第一版即采用收费订阅模式。软件本体可以免费下载和安装，但核心 AI 调用、金融模板、内置 skills、长任务能力和报告生成能力按订阅等级开放。

订阅等级建议控制在 2-3 档，避免早期定价过于复杂。

### 13.1 Research

面向个人投资者、轻度量化研究者和早期试用用户。

核心权益：

- 基础 token 月额度
- 标准模型调用
- 基础文件 / bash / web / plan / skill 能力
- 基础因子检测脚本
- 基础策略回测脚本
- 简版因子检测报告
- 简版策略回测报告
- session 持久化
- 基础安全确认

适合场景：

- 验证简单因子
- 运行小规模回测
- 分析单篇研报
- 进行一般投资问题咨询

### 13.2 Pro

面向高频使用的个人研究者、职业投资者和量化开发者。

核心权益：

- 更高 token 月额度
- 可使用更强模型
- 更长上下文
- 更多内置金融 skills
- 更完整的因子检测脚本
- 更稳定的策略回测脚本
- 标准化 HTML / Markdown 报告生成
- 策略迭代 plan 模板
- 研报复现基础能力
- 更好的 tool 生命周期展示
- token 使用展示

适合场景：

- 多因子检测
- 策略持续迭代
- 研报策略复现
- 中等规模数据回测
- 生成稳定格式研究报告

### 13.3 Team

面向小型量化团队、投研团队和专业机构。

核心权益：

- 团队共享 token 池
- 更高并发或更高任务额度
- 团队成员管理
- 团队共享 skills
- 团队共享脚本模板
- 更稳定的报告模板
- 团队级安全策略
- 项目级 policy
- 内部数据源适配能力
- 团队账单和使用统计

适合场景：

- 团队协作研究
- 多策略并行迭代
- 统一研究报告格式
- 团队内部策略模板沉淀
- 私有数据源接入

### 13.4 等级区分维度

订阅等级主要通过以下维度区分：

- token 月额度
- 可用模型等级
- 上下文长度
- 内置金融 skills 数量
- 内置回测 / 报告脚本数量
- 报告生成稳定性
- 长任务额度
- 团队协作能力

具体价格和 token 额度可后续根据模型成本、目标客群和使用强度再确定。

## 14. 产品原则

ChainPeer 应坚持：

- 金融场景优先
- 本地优先
- 安装简单
- 使用透明
- 成本可见
- 执行可控
- 长任务可追踪
- 结果可复现
- 能力可扩展

最终目标是：让用户感觉它不是一个普通聊天机器人，而是一个能真正参与金融研究、策略开发和投资分析工作的本地 AI Agent。
