# Agent 项目下一步完善建议（审阅版）

## 1) 当前结构简评

你的项目已经具备一个可用的 Agent 雏形，核心链路是完整的：

- **入口层**：`main.py` 启动 CLI；
- **Agent 层**：`basic_agent.py` 负责对话循环、工具调用、流式输出；
- **Prompt 层**：`agent/prompts.py` 定义系统策略；
- **工具层**：`tools/` 已覆盖 `read/write/edit/grep/bash/web`；
- **体验层**：`utils/logo.py` 已做命令行品牌化展示。

这已经超过“玩具 demo”，接下来建议从“可控性、可靠性、可维护性”三个方向升级。

---

## 2) 优先级最高（P0，建议先做）

## P0-1 工具调用安全与容错

### 问题
- 当前 `BasicAgent._execute_tool()` 直接执行工具函数，缺少统一异常包装；
- `bash` 工具缺少危险命令保护（如 `rm -rf`, `del /s` 等）；
- 工具返回值目前全是字符串，不利于后续自动判断“成功/失败”。

### 建议
- 增加统一 `safe_execute_tool()`：捕获异常并返回结构化结果；
- 给 `bash` 增加基础命令黑名单/确认策略；
- 工具输出升级为统一结构（如 `{"ok": bool, "data": ..., "error": ...}`）。

### 验收标准
- 任一工具抛异常时，Agent 不崩溃，且能继续对话；
- 危险命令默认被拦截并给出明确理由；
- Agent 能依据结构化字段判断下一步动作。

---

## P0-2 上下文管理（避免长对话退化）

### 问题
- `chat_history` 只增不减，长会话会导致延迟升高、费用增加、回答质量下降。

### 建议
- 实现“最近 N 轮 + 压缩摘要”的上下文策略；
- 设定 token budget，超过阈值自动压缩历史对话；
- 保留工具调用结果摘要，而不是完整原始文本。

### 验收标准
- 长会话（>50轮）仍可稳定交互；
- 平均响应延迟和 token 使用可控。

---

## P0-3 观测与调试能力（可定位问题）

### 问题
- 目前调试信息主要靠 `print`，难以复盘；
- 缺少“每轮调用了哪些工具、耗时多少、是否失败”的日志。

### 建议
- 增加结构化日志（JSONL）输出到 `logs/`；
- 每轮记录：用户输入、模型响应、工具调用、耗时、异常；
- 增加 `--trace` 模式，用于问题复现。

### 验收标准
- 发生异常时能在日志中快速定位到具体工具和参数；
- 可追踪一次完整对话链路。

---

## 3) 中优先级（P1，稳定后推进）

## P1-1 测试体系（单测 + 集成）

### 建议
- 建立 `pytest` 基础；
- 先覆盖高价值模块：`file_ops.py`、`bash.py`、`web.py`；
- 增加最小集成测试：模拟一次“问答 -> 工具调用 -> 回答”链路。

### 重点场景
- `edit_file` 多匹配拒绝；
- `grep` 正则异常与大文件跳过；
- `bash` 超时与 `cd` 状态保持；
- `search_web/fetch_web_page` 网络失败降级。

---

## P1-2 Web 工具可靠性与安全性

### 问题
- 目前 `web.py` 使用 `verify=False`，可用性上可行，但安全性和一致性一般；
- 搜索依赖 HTML 解析，页面结构变化后易失效。

### 建议
- 增加可配置 `VERIFY_SSL` 开关（默认开启，必要时关闭）；
- 搜索结果解析加入多选择器兜底；
- 加入请求重试和退避（例如 429/5xx）。

---

## P1-3 Prompt 与工具能力自动对齐

### 问题
- prompt 文本和工具清单是手工同步，后续容易漂移。

### 建议
- 从 `TOOL_SCHEMAS` 自动生成“工具能力片段”并注入系统提示；
- 避免“文档里有，代码里没”或反之。

---

## 4) 低优先级但很加分（P2）

## P2-1 会话持久化与恢复
- 支持保存/加载历史会话（如 `sessions/*.json`）；
- 启动时可选择继续上一次上下文。

## P2-2 多 Agent/子任务执行
- 主 Agent 负责规划，子 Agent 执行只读检索任务；
- 适合复杂代码库分析和批量任务。

## P2-3 CLI 体验升级
- 增加 `/help`, `/tools`, `/clear`, `/save`, `/load` 指令；
- 响应结束后显示耗时与 token 统计。
